# The Priority Queue — 우선순위 큐 할 일 관리자

> 소프트웨어공학 팀프로젝트 / 주제 8 — "공학적 포인트: Queue/Heap 자료구조"

"다음에 무엇을 할까"라는 결정을 사람이 매번 내리지 않도록, **이진 최대-힙(Binary Max-Heap)** 이
O(log n)으로 그 자리를 대신하는 할 일 관리자입니다. 그리고 **그 판단의 근거까지** 보여줍니다.

단일 HTML 파일이던 프로토타입을 **Vite + React + TypeScript** 정식 프로젝트로 재구성하고,
핵심 로직을 모듈로 분리해 **유닛 테스트로 정확성을 증명**했습니다.

## 핵심 공학 포인트

### 1. 자료구조 — 직접 구현한 이진 최대-힙 (`src/lib/MaxHeap.ts`)
`Array.prototype.sort()` 를 쓰지 않고 힙을 직접 구현했습니다.

| 연산 | 복잡도 | 구현 |
|------|--------|------|
| `insert` | O(log n) | siftUp (부모와 비교하며 위로) |
| `extractMax` | O(log n) | siftDown (큰 자식과 교환하며 아래로) |
| `heapsort` | O(n log n) | 최댓값 반복 추출로 완전 정렬 |

배열 표현: `a[i]` 의 부모 = `a[(i-1)/2]`, 자식 = `a[2i+1]`, `a[2i+2]`.
힙 속성: 모든 부모 노드 점수 ≥ 두 자식 점수.

### 2. 점수 공식 (`src/lib/scoring.ts`)
```
score = 중요도(1~4) × 긴급도(1~5) + 가중치(0~2)      // 이론상 최대 22점
```
긴급도는 마감일까지 남은 일수로 환산합니다 (오늘/초과=5 … 일주일+=1).

### 3. Strategy 패턴 ×2
- **분석기 전략** (`src/lib/analyzers.ts`): `Analyzer` 인터페이스로 추론 전략을 교체 가능하게
  설계. 현재는 외부 의존성 0인 **규칙기반(키워드 매칭)** 전략을 제공.
- **에너지 필터 전략** (`src/lib/energyStrategies.ts`): 현재 컨디션(전체 / 몰입 / 저전력)에 따라
  보여줄 할 일을 거르는 전략.

> 이전 프로토타입의 "Claude API" 분석 전략은 제거했습니다. 브라우저에서 Anthropic API를 직접
> 호출하는 구조라(키 노출·CORS) 실제로는 항상 규칙기반으로 폴백되어, 동작하지 않는 버튼이었기
> 때문입니다. `Analyzer` 인터페이스는 그대로라, 추후 서버 프록시 전략을 한 곳에만 끼워 넣으면 됩니다.

## 화면 구성 — "사람의 뷰"와 "기계의 뷰"

- **사람의 뷰 (왼쪽)**: heapsort 로 완전 정렬된 우선순위 목록. 각 카드는 *왜 이 순위인지* 설명합니다.
- **기계의 뷰 (오른쪽, 인스펙터)**: 메모리에 실제 저장된 힙 구조를 트리·배열·연산 로그로 시각화.
  - **현재 상태** 탭: 라이브 힙, 노드 수·트리 높이·비교 횟수·루트 값.
  - **단계별 시연** 탭: 임의 값을 `insert` 하거나 `extractMax` 했을 때 siftUp·siftDown 의
    비교/교환을 **한 스텝씩 애니메이션**으로 재생 (실제 큐는 건드리지 않는 안전한 데모).

## 주요 기능

- 규칙기반 분석으로 중요도·가중치·에너지·점수 자동 추론 → 확인/수정 후 큐에 삽입
- **힙 연산 단계별 시각화** (siftUp / siftDown 재생)
- 할 일 **수정**(중요도·마감·가중치·에너지) / **검색·필터** / 완료·삭제
- 컨디션 기반 **에너지 모드**(전체 / 몰입 / 저전력)
- **통계 대시보드**: 완료율, 중요도/에너지 분포, 임박 항목, 평균·최고 점수
- **일반 / 재미 모드 토글** — 재미 모드에서는:
  - **NOW SERVING** 카드로 큐의 맨 앞(최우선)을 띄우고 `다음 처리(dequeue)` 버튼으로 루트를 pop
  - 추가·완료·정렬 시 오른쪽 힙 트리의 노드가 **실시간으로 제자리로 미끄러지는** 애니메이션
  - 가벼운 **사운드**(Web Audio 합성)와 **컨페티**(의존성 없는 캔버스 파티클)
- `localStorage` 자동 저장 (모드·할 일 모두 / 접근 불가 환경에선 in-memory 폴백)

## 시연 가이드 (3분 데모)

1. **추가 → 우선순위 자동 결정**: 입력창에 `내일까지 교수님께 발표자료 제출` → "분석 후 추가".
   규칙기반이 중요도 4·가중치 +2를 제안 → 마감일을 내일로 잡고 삽입 → 점수 18로 **1순위** 등극.
2. **근거 확인**: 각 카드 아래 "왜 이 순위인지" 문장과 점수 막대를 보여주기.
3. **기계의 뷰 → 단계별 시연 탭**: 점수 `20` 을 `insert` → siftUp 으로 루트까지 올라가는 과정을
   "다음"으로 한 스텝씩, 혹은 "재생"으로 자동 재생. 이어서 `extractMax` 로 siftDown 시연.
4. **에너지 모드**: "몰입 모드"로 전환 → 집중이 필요한 일만 남는 것을 확인.
5. **통계 탭**: 완료율·분포를 한눈에. 몇 개를 완료 처리하면 완료율이 즉시 갱신.

## 프로젝트 구조

```
src/
├── lib/                  # 순수 로직 (테스트 대상)
│   ├── MaxHeap.ts        # 이진 최대-힙 + 단계별 trace 함수
│   ├── scoring.ts        # 점수·긴급도·D-day 계산
│   ├── analyzers.ts      # Strategy #1: 규칙기반 분석기
│   ├── energyStrategies.ts # Strategy #2: 에너지 필터
│   └── *.test.ts         # Vitest 유닛 테스트 (37 cases)
├── hooks/
│   └── useTasks.ts       # 상태 + localStorage 영속화 + CRUD
├── components/           # UI (프레젠테이션)
│   ├── Masthead, Composer, DraftCard, TaskCard, SearchBar,
│   ├── EditModal, Inspector, HeapTree, StepThrough,
│   └── StatsDashboard, Toast, icons
├── data/seed.ts          # 시연용 초기 데이터
├── types.ts              # 도메인 타입
├── App.tsx               # 오케스트레이션
└── main.tsx              # 진입점
```

**관심사 분리**: `lib/` 의 로직은 React·DOM에 의존하지 않는 순수 함수/클래스라
Node 환경에서 그대로 테스트됩니다. UI는 그 위에 얇게 얹혀 있습니다.
